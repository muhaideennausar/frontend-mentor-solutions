# Frontend Mentor - Results summary component solution

This is my solution to the **Results summary component** challenge from Frontend Mentor. The goal of this project was to build a clean and responsive results summary card using only HTML and CSS.

## Overview

### The challenge

Users should be able to:

- View the optimal layout for the interface depending on their device's screen size
- See hover and focus states for interactive elements

## My process

### Built with

- Semantic HTML5 markup
- CSS custom properties
- Flexbox
- Mobile-first workflow
- JavaScript
- Local JSON data

### What I learned

I practiced rendering UI content dynamically from a local JSON file using JavaScript. I also used array methods like `map()` to generate markup and `reduce()` to calculate the overall score.

### Continued development

In future iterations, I’d like to improve accessibility, add stronger focus states, and make the result text more configurable based on score ranges.
